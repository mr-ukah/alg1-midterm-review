Expressions Writing, Rewriting, and Breaking Apart Algebraic Expressions
Consecutive Integers
Writing expressions for "the next integer" or "the next odd integer"
Used in: Session 1 · Q3
The core idea
What "consecutive" means

Consecutive integers are integers that come one after another with no gaps — like 4, 5, 6 or −3, −2, −1. If the first one is n, the next one is always n + 1.

Consecutive integers: n, n+1, n+2, ... Example: if n = 7, the next is 8, then 9
Consecutive ODD integers

Odd integers skip every other number — like 3, 5, 7 or −1, 1, 3. The gap between consecutive odd integers is always 2, not 1. So if the first odd integer is n, the next odd integer is n + 2.

Consecutive odd integers: n, n+2, n+4, ... Example: if n = 5 (odd), next odd is 7 = 5+2 Their product: n(n+2)
⚠️
Common mistake: Writing n(n+1) for consecutive odd integers. That gives you any two consecutive integers (like 5 and 6), not two consecutive odd integers (like 5 and 7). The gap between odds is 2, so you need n(n+2).
Why n+1 vs n+2 — a visual check
Consecutive integers (any): gap of 1 4 → 5 → 6 → 7 Expression: n and n+1
Consecutive ODD integers: gap of 2 3 → 5 → 7 → 9 Expression: n and n+2
Consecutive EVEN integers: gap of 2 4 → 6 → 8 → 10 Expression: n and n+2
💡
Quick check: Plug in a real odd number. If n = 3, does your expression give the next odd? n+1 = 4 — that's even, so wrong. n+2 = 5 — that's the next odd. ✓
Exponent Rules
Rewriting expressions that multiply powers of the same base
Used in: Session 2 · Q7
The rule you need
When you multiply two powers with the same base, add the exponents

Think of it this way: 8² means "8 times itself twice." 8³ means "8 times itself three times." Multiplying them gives you 8 times itself five times total — so 8⁵. You're just counting up the total number of times you multiply.

bᵐ · bⁿ = b^(m+n) Same base? Add the exponents. Example: 8² · 8³ = 8^(2+3) = 8⁵
Applying this to 8 · 8^a

The plain number 8 is the same as 8¹ — any number to the first power is itself. So 8 · 8^a = 8¹ · 8^a = 8^(1+a) = 8^(a+1).

⚠️
Common mistake: Adding to the base instead of the exponent. 8 · 8^a is NOT 16^a (you didn't change the base) and NOT 8^(8a) (you don't multiply exponents here — that's for a power raised to a power). You're multiplying, so you ADD exponents.
Step by step
Given: x = 8^a. Find 8x.
Step 1: Substitute x = 8^a 8x = 8 · 8^a
Step 2: Rewrite 8 as 8¹ = 8¹ · 8^a
Step 3: Same base — add exponents = 8^(1+a) = 8^(a+1)
💡
Check with numbers: If a = 2, then x = 8² = 64, and 8x = 8(64) = 512. Does 8^(a+1) = 8³ = 512? Yes. ✓
Factoring
Breaking a polynomial into a product of simpler expressions
Used in: Session 3 · Q9
Three factoring moves, in the order you always try them
STEP 1
Always pull out the GCF first
The GCF (greatest common factor) is the biggest thing that divides evenly into every term. Factor it out front before doing anything else. Missing the GCF is the most common factoring error.
STEP 2
Count the terms — then choose your method
2 terms: Look for difference of squares — a² − b² = (a+b)(a−b). 3 terms (trinomial): Find two numbers that multiply to c and add to b. 4 terms: Try grouping.
STEP 3
Check each factor — can it be factored further?
"Completely factored" means none of the remaining pieces can be broken down further. Always double-check.
Worked example: −x³ + 10x² + 24x
Step 1: GCF Every term has an x. The leading term is negative, so pull out −x: −x³ + 10x² + 24x = −x(x² − 10x − 24)
Step 2: Factor the trinomial x² − 10x − 24 Need two numbers that multiply to −24 and add to −10. Try: −12 and +2 → product = −24 ✓, sum = −10 ✓ x² − 10x − 24 = (x − 12)(x + 2)
Full factored form: −x(x − 12)(x + 2)
⚠️
Sign trap — factoring out a negative: When you factor out −x, every sign inside flips. −x³ becomes +x², and +24x becomes −24. Write it out carefully before moving to step 2.
Finding the two numbers for a trinomial
For x² + bx + c, find p and q where: p · q = c AND p + q = b
List factor pairs of c, check which pair sums to b. Example: x² − 10x − 24, so c = −24, b = −10 Factor pairs of −24 (one +, one −): +1, −24 → sum = −23 ✗ +2, −12 → sum = −10 ✓ +3, −8 → sum = −5 ✗ Answer: (x + 2)(x − 12)
💡
Always check by multiplying back (FOIL): (x+2)(x−12) = x² − 12x + 2x − 24 = x² − 10x − 24. ✓

How to read interval notation
Parenthesis ( ) vs bracket [ ]

A parenthesis means the endpoint is not included (open circle on a number line). A bracket means the endpoint is included (closed circle). Infinity always gets a parenthesis — you can never actually reach infinity.

(3, ∞) → x > 3 (3 not included) [3, ∞) → x ≥ 3 (3 is included) (1, 5) → 1 < x < 5 (neither included) [1, 5] → 1 ≤ x ≤ 5 (both included)
⚠️
∞ always gets a parenthesis, never a bracket. You can never "include" infinity — it's not a real number, just a direction. Writing [3, ∞] is always wrong.
How to read set-builder notation
Reading {x | condition}

The curly braces { } mean "the set of all..." The letter before the | (vertical bar, read "such that") names what you're collecting. The condition after | tells you the rule.

{x | x > 5} "The set of all x such that x is greater than 5" {x | x ≥ 0} "The set of all x such that x is 0 or greater" {x | 1 < x ≤ 10} "All x greater than 1 and at most 10"
💡
All three say the same thing. x > 5, (5, ∞), and {x | x > 5} are three different ways to write exactly the same idea. Your teacher may ask you to convert between them — practice going both directions.
Linear Systems
Finding where two lines cross — and checking statements about that intersection
Used in: Session 2 · Q2 (wait — Q2 is Session 3). Session 2 · Q8 (point on a line). Session 3 · Q2.
What a solution to a system means
Big idea

A system of two linear equations is just two lines. The solution is the point where they cross — the one (x, y) pair that makes both equations true at the same time.

How to solve by substitution

1. Solve one equation for y (or x). 2. Substitute that expression into the other equation. 3. Solve for the remaining variable. 4. Plug back in to find the other.

⚠️
Evaluating true/false statements about a system: Check each statement individually. "f(x) = g(x) at x = 4" means: plug x = 4 into both and see if the outputs match. "The graphs intersect at (4, 12)" means: x = 4 AND y = 12 must satisfy both equations.
Worked example: f(x) = (2/3)x − 4 and g(x) = (1/4)x + 1
Find where they intersect: set equal (2/3)x − 4 = (1/4)x + 1 Multiply everything by 12 (LCM of 3 and 4): 8x − 48 = 3x + 12 5x = 60 → x = 12 Find y: f(12) = (2/3)(12) − 4 = 8 − 4 = 4 Intersection: (12, 4)
Check statement "f(4) = g(4)": f(4) = (2/3)(4)−4 = 8/3−4 = −4/3 g(4) = (1/4)(4)+1 = 2 ≠ −4/3 → FALSE
Literal Equations
Solving a formula for a different variable — rearranging the formula
Used in: Session 2 · Q13
The idea
A literal equation is a formula with multiple variables

"Solving for F" means getting F alone on one side. You use the exact same moves as regular equation solving — add, subtract, multiply, divide on both sides — but instead of getting a number at the end, you get an expression in terms of the other variables.

K = (5/9)(F + 459.67) Goal: isolate F Step 1: Multiply both sides by 9/5 (9/5)K = F + 459.67 Step 2: Subtract 459.67 from both sides F = (9/5)K − 459.67
⚠️
Undo in reverse order. In the original formula, the operations applied to F were: add 459.67, then multiply by 5/9. To undo, reverse the order: first multiply by 9/5 (undo the multiplication), then subtract 459.67 (undo the addition).
Step by step
K = (5/9)(F + 459.67)
Multiply both sides by 9/5: (9/5)K = F + 459.67
Subtract 459.67 from both sides: F = (9/5)K − 459.67
Check: plug F back in. If K=273.15 (freezing in Kelvin): F = (9/5)(273.15) − 459.67 = 491.67 − 459.67 = 32°F ✓
💡
Treat the other variables like numbers. K is just a placeholder for "whatever the Kelvin value is." Treat it exactly like you would treat a number — multiply, divide, add, subtract it as needed.
Systems & Graphing
Writing a system from a story, graphing both lines, and interpreting where they cross
Used in: Session 3 · Q18
Writing the system
Step 1: Identify what x represents

The problem will tell you. Usually x = number of years, weeks, months, etc. since a starting point. Define it clearly before writing any equation.

Step 2: Write y = mx + b for each quantity

The starting amount is b (the y-intercept — what you have when x = 0). The rate of change is m (the slope — how much it goes up or down per unit of x).

Ski team: 5 members in 2010, +10/year y = 10x + 5 Chorus: 35 members in 2010, +5/year y = 5x + 35
Step 3: Find the intersection

Set the two expressions equal and solve. The x-value tells you when the two quantities are equal. The y-value tells you what both are equal to at that moment.

💡
Interpreting the intersection point: Every coordinate has a real-world meaning. If the intersection is (6, 65), that means "after 6 years, both groups have 65 members." Say both things — the x meaning AND the y meaning.
Graphing the system
For each line, find two easy points: y-intercept: set x = 0 One more: pick a convenient x
y = 10x + 5: x=0: y=5 → (0, 5) x=6: y=65 → (6, 65) y = 5x + 35: x=0: y=35 → (0, 35) x=6: y=65 → (6, 65)
Both pass through (6, 65) → intersection ✓
⚠️
Label your graph. Write what each line represents next to it. The grader needs to see which line is which — a graph with two unlabeled lines earns less credit than one with labels.
Functions Understanding and Describing How Inputs Relate to Outputs
Notation Reference
Set-builder notation and domain — describing which inputs a function accepts
What domain means
Big idea

The domain of a function is the set of all inputs (x-values) that make sense. Think of it as the list of all x-values you're allowed to plug in.

Writing domain in set-builder notation

{x | condition} means "all x-values where [condition] is true." The | is read "such that."

{x | x ≥ 0} "All x such that x is 0 or greater" {x | x is a non-negative integer} "x can be 0, 1, 2, 3, ..." (discrete)
⚠️
Discrete vs. continuous domain: Some functions only make sense for whole numbers (discrete) — like number of ice cream cones sold. Others make sense for any real number (continuous) — like temperature over time. The domain must reflect what makes sense in context, not just what's mathematically possible.
Comparing the notations for domain
SituationSet-builderInterval

x ≥ 0 (continuous){x | x ≥ 0}[0, ∞)
Whole numbers ≥ 0{x | x ∈ ℤ, x ≥ 0}n/a (discrete)

💡
Interval notation only works for continuous domains (ranges of real numbers). For discrete domains (only integers, only whole numbers), use set-builder notation or describe it in words.
Sequences
Arithmetic sequences (add the same amount each time) vs. exponential sequences (multiply each time)
Used in: Session 1 · Q5 · Session 2 · Q15
Arithmetic sequences
What makes it arithmetic

You get from one term to the next by adding the same number every time. That number is called the common difference. To find it: subtract any term from the term that follows it.

2a+1, 4a+4, 6a+7, 8a+10 Subtract term 1 from term 2: (4a+4) − (2a+1) = 2a+3 Check term 2 to term 3: (6a+7) − (4a+4) = 2a+3 ✓ Common difference = 2a+3
⚠️
Subtract in order: Always do (later term) − (earlier term), not the other way around. Getting the sign wrong gives you the negative of the common difference.
Exponential sequences
What makes it exponential

You get from one term to the next by multiplying by the same number every time. That number is called the common ratio. To find it: divide any term by the term before it.

1, 3, 9, 27, 81, ... Check differences (arithmetic test): 3−1=2, 9−3=6, 27−9=18 → NOT constant ✗ Check ratios (exponential test): 3/1=3, 9/3=3, 27/9=3 → constant ratio ✓ Common ratio = 3 → exponential sequence
💡
Two-step check: (1) Are the differences constant? If yes → arithmetic. (2) Are the ratios constant? If yes → exponential. If neither → neither. Exponential growth gets much faster over time; arithmetic growth stays steady.
Functions & Domain
What makes something a function — and what inputs it accepts
Used in: Session 1 · Q10 · Session 2 · Q11
The function rule
A function has exactly one output for every input

Think of a function like a machine: you put one number in, you get exactly one number out. The rule is: one x-value cannot map to two different y-values. If it does, it's not a function.

The vertical line test (for graphs)

Draw (or imagine) a vertical line anywhere on the graph. If that vertical line ever touches the graph in more than one place, it's NOT a function. The graph is "curving back" — meaning one x is giving two y-values.

⚠️
A curve can curve — but not back on itself horizontally. A curve that goes up and then back down (like a parabola) passes the vertical line test. A curve that goes right and then back left fails it.
Domain: what inputs are allowed
Context determines domain

Even if a formula works for any number mathematically, the real-world situation may limit what makes sense. Always ask: what values of x are realistic for this situation?

P(c) = 0.87c (profit per cone) Mathematically: c can be any real number In context: c = cones sold → can't sell a fraction of a cone → can't sell a negative number of cones Domain: {c | c ∈ ℤ, c ≥ 0} (integers greater than or equal to zero)
💡
Discrete vs. continuous: Cones sold = discrete (only whole numbers). Temperature over time = continuous (any real number in a range). The answer choices "integer" vs. "rational number" test whether you know this distinction.
Transformations
Shifting graphs up, down, left, and right — and why the direction can be counterintuitive
Used in: Session 3 · Q12
The four basic shifts
What you writeWhat it does to the graph
f(x) + kShifts UP by k units
f(x) − kShifts DOWN by k units
f(x − h)Shifts RIGHT by h units
f(x + h)Shifts LEFT by h units
⚠️
Horizontal shifts are backwards from what you expect. f(x − 3) shifts RIGHT (not left), and f(x + 3) shifts LEFT (not right). Here's why: f(x − 3) = 0 when x = 3, so the "zero point" moved to the right. Think of it as: the graph moves in the opposite direction of the sign inside the parentheses.
Applying this to |x|
Original: h(x) = |x| (V-shape, vertex at origin)
Shift RIGHT 3: replace x with (x − 3) g(x) = |x − 3| Vertex moves from (0,0) to (3,0)
Shift UP 3: add 3 outside g(x) = |x| + 3 Vertex moves from (0,0) to (0,3)
Shift LEFT 3: replace x with (x + 3) g(x) = |x + 3| Vertex moves from (0,0) to (−3,0)
💡
Quick check for horizontal shifts: Ask yourself "what value of x makes the expression inside equal to zero?" That x-value is the new vertex location. For |x − 3|, x = 3 makes it zero → vertex at x = 3 → shifted right 3. ✓
Models Using Functions to Describe Real-World Situations
Linear Models
Writing and interpreting linear functions from context and from tables
Used in: Session 2 · Q8 · Session 3 · Q16a
Slope and y-intercept — what they mean in context
The slope (m) is the rate of change

Slope tells you how much y changes for every 1-unit increase in x. In context: "per bracelet," "per mile," "per year." It's always a rate.

The y-intercept (b) is the starting value

The y-intercept is the value of y when x = 0. In context: the cost before you buy anything, the amount saved before you earn anything, the starting population.

Finding slope from a table

Pick any two rows. Slope = (change in y) ÷ (change in x) = (y₂ − y₁) ÷ (x₂ − x₁). Then verify with another pair — if it's linear, every pair gives the same slope.

⚠️
Don't assume the y-intercept is in the table. If x = 0 isn't listed, you need to calculate b using y = mx + b with a known point. Plug in the slope and one (x, y) pair and solve for b.
Worked example: bracelets table
Table: x=4→7.50, x=6→9.00, x=9→11.25, x=10→12.00
Step 1: Find slope using (4, 7.50) and (6, 9.00) m = (9.00 − 7.50) / (6 − 4) = 1.50 / 2 = 0.75
Step 2: Find b using y = 0.75x + b with (4, 7.50) 7.50 = 0.75(4) + b = 3 + b b = 4.50
Step 3: Write the function f(x) = 0.75x + 4.50
Meaning: slope 0.75 = $0.75 per bracelet y-intercept 4.50 = $4.50 fixed starting cost
Exponential Models
Writing and using exponential functions for growth and decay
Used in: Session 1 · Q16b
The exponential model formula
General form: V(t) = a · b^t

a is the starting value (what you have when t = 0). b is the multiplier each time period. If b < 1, the value shrinks (decay/depreciation). If b > 1, the value grows.

Depreciation rate = 14.5% per year The car LOSES 14.5% each year It KEEPS 100% − 14.5% = 85.5% each year b = 0.855 (not 0.145 — that's what's lost)
⚠️
Common mistake: Using b = 0.145 (the depreciation rate) instead of b = 0.855 (what remains). The base b is always what you KEEP, which is 1 minus the rate that's lost.
Worked example: car depreciation
Car purchased for $12,000. Depreciates 14.5%/year.
Write the function: V(t) = 12000 · (0.855)^t
Find depreciation from year 3 to year 4: V(3) = 12000 · (0.855)³ = 12000 · 0.625... ≈ $7,513.77 V(4) = 12000 · (0.855)⁴ = 12000 · 0.534... ≈ $6,424.27 Depreciation = V(3) − V(4) ≈ $1,089.50
(Use a calculator — round only at the final step)
💡
Don't round intermediate steps. Keep full decimal precision until you get to the final subtraction. Rounding V(3) and V(4) separately before subtracting can throw off your final answer by several dollars.
Quadratic Models
Minimum values, projectile motion, and the discriminant
Used in: Session 1 · Q14 · Session 2 · Q1 · Session 3 · Q4
Finding the minimum value
From an equation: use the vertex formula

For f(x) = ax² + bx + c, the minimum (if a > 0, parabola opens up) is the y-value at the vertex. Find it with: x = −b/(2a), then plug that x back into f(x).

From a table: find the smallest y-value

Scan the output column. The minimum is just the lowest number in the f(x) or r(x) column. In the r(x) table, the lowest value is −16, occurring at x = −2.

⚠️
Compare the minimum VALUES, not the x-values. The question asks which function has the smaller minimum value — that means compare the y-values at each vertex, not the x-coordinates.
The discriminant — rational vs. irrational roots
What the discriminant tells you

For ax² + bx + c = 0, the discriminant is b² − 4ac. It lives under the square root in the quadratic formula. If it's a perfect square (0, 1, 4, 9, 16, 25, ...) → roots are rational. If it's not a perfect square → roots are irrational (they have a square root that doesn't simplify to a whole number).

2x² + 3x − 10 = 0 b² − 4ac = 9 − 4(2)(−10) = 9 + 80 = 89 89 is not a perfect square → roots are irrational
Projectile motion — solving for a specific height
h(t) = −16t² + 5t + 105 Question: when does diver fall 45 feet below start? Starting height = 105. Fall 45 ft → new height = 60.
Set h(t) = 60: −16t² + 5t + 105 = 60 −16t² + 5t + 45 = 0 16t² − 5t − 45 = 0
Quadratic formula: a=16, b=−5, c=−45 t = (5 ± √(25 + 2880)) / 32 t = (5 ± √2905) / 32 t ≈ (5 + 53.9) / 32 ≈ 1.84 seconds (take the positive root — time can't be negative)
⚠️
"45 feet below the starting point" ≠ height of 45. Starting height is 105, so 45 feet below is 105 − 45 = 60. Set h(t) = 60, not h(t) = 45. Reading the question carefully is half the problem.
